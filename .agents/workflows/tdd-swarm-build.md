# Workflow: TDD Swarm Build & Automated Marketing Capture
**Description**: Orchestrates a multi-agent swarm to build a feature, verify it via browser-in-the-loop testing, and automatically format the testing artifact into a marketing video for offline review.

## Step 1: Feature Implementation (Architect & CDO)
- **Action**: Spawn the Architect and/or Chief Web Design Officer (CDO) to build the requested feature.
- **Task**: Write the Svelte 5 logic and enforce the "Nuclear Americana Tech Noir" aesthetic. 
- **Constraint**: Must strictly adhere to the 80-line function limit and Zero-Trust payload stripping defined in `@GEMINI.md`.

## Step 2: Browser-in-the-Loop Validation (CRO)
- **Action**: Spawn the Chief Reliability Officer (CRO) agent.
- **Task**: Launch the local development server and execute the end-to-end test suite using Puppeteer. 
- **Task**: The CRO must physically click through the new UI component to verify reactivity and capture the session as a high-quality browser recording Artifact.

## Step 3: Marketing Video Generation (CMO)
- **Action**: Spawn the Chief Marketing Officer (CMO) agent.
- **Task**: Take the raw browser recording Artifact generated by the CRO in Step 2.
- **Task**: Format, crop, and optimize the video for web performance (e.g., converting to webm or mp4 with appropriate aspect ratios).
- **Task**: Save the finalized promotional video strictly to the `/marketing/pending-review/` directory.
- **CRITICAL CONSTRAINT**: The CMO is mathematically prohibited from executing any deployment, API, or upload scripts. The video must remain local for human offline review. 

## Step 4: Human-in-the-Loop Review
- **Action**: Halt the swarm execution.
- **Output**: Notify the user via the chat interface that the feature has passed testing and the marketing video is ready. Instruct the user to open the Auxiliary Pane or check the `/marketing/pending-review/` directory to view the video offline.
