# PROJECT PHOENIX (SSTRACKER) - THE BILLION-DOLLAR ROADMAP

## Product Vision & Ecosystem

North-star **Core Ecosystem Directives** for every product decision, feature, and sprint. All future development should trace back to these pillars. 

### 1. The Ultimate Metric (Player & Parent First)
Every feature must be evaluated through this lens:
- **Does it increase a player’s physical engagement and skill development?**
- **Does it provide tangible ROI and security peace-of-mind for the parent?**
Administrative ease is secondary to user growth. Staff tooling matters, but it must not outrank player/parent value when priorities collide.

### 2. The Nexus Vanguard Standard (UI/UX VFX Mandate)
We are building a multi-billion-dollar Enterprise SIEM wrapped in a AAA gaming HUD (Iron Man / Destiny 2). 
- **Zero Web 2.0 Components:** Native scrollbars, white boxes, and basic HTML inputs are permanently banned.
- **VFX Compositing:** All SVGs (tactical boards, radars) must utilize glowing bloom filters (`feGaussianBlur`) and precision nodes.
- **Militant Geometry:** All dashboards operate on a strict, mathematical grid (e.g., the 4-4-4 Command Grid).

### 3. Player OS (Weaponized Gamification)
Implement deep RPG progression mechanics driven by a polynomial math curve (Cap 99). At-home physical training must be converted into **XP**, unlocking ranks, visual ID card upgrades, and Quartermaster currency. 

### 4. Coach OS (Tactical Telemetry)
Move beyond static scheduling. Create automated feedback loops where game-day statistical deficiencies logged via the **Sideline Telemetry HUD** seamlessly generate at-home training **Bounties**.

### 5. Parent OS (ROI & Security)
Provide parents with visual **Skill Trees** tracking their athlete’s development, wrapped in an impenetrable, zero-trust COPPA compliance vault (The Triad Protocol).

### 6. Agnostic Multi-Sport Architecture
The platform is sport-agnostic. All schemas, loggers, and boards dynamically populate from a **5-Pillar Sport Payload**:
1. **Taxonomy** — Positions and roles.
2. **Telemetry Dictionary** — Game stats and event vocabulary.
3. **Combine Metrics** — Recruiting baselines.
4. **Tactical Assets** — Pitch/court SVGs and board chrome.
5. **Baseline Workout Libraries** — Sport-specific drills wired into the gamification engine.

## Phase 6: The Automated Dispatch Engine (AI Coach)
**Mission profile:** The platform becomes a tactical co-pilot. Every logged workout is a signal; the system responds with targeted, standards-aligned homework.
- **The Gen 2 Cloud Function:** `onWorkoutLogged` evaluates submissions against performance thresholds.
- **Automated Dispatch:** Injects a targeted homework payload into the athlete’s Action Inbox.
- **The Command Override Protocol (NON-NEGOTIABLE):** Human Coach curriculum is sovereign. The AI must never overwrite a human-locked curriculum. If an auto-dispatch adds value outside the locked curriculum, it routes to the Coach for one-click approval.
